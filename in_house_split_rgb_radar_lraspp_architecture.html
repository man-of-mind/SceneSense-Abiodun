Block diagram for the CARLA traffic-light-pole split-inference model: RGB and radar are fused at the sensor side, MobileNetV3 LR-ASPP backbone features are compressed and sent over UDP, and the network-side tail reconstructs features to produce segmentation and object-localization output.

UDP Downlink

Server sends {mask, objects, server_ms} back to camera result port 51004.

Neural Layers

  • The sender-side neural work stops after the MobileNetV3 LR-ASPP backbone.
  • The adapted first convolution accepts RGB plus four radar raster channels.
  • The segmentation branch uses LR-ASPP features to output three semantic classes.
  • The object branch outputs an 11-channel dense map: center heatmap plus local XYZ, dimensions, yaw sin/cos, parked score, and radar support.

Transport Path

  • serialize_feature_maps() encodes each backbone feature tensor before uplink.
  • Default transport uses per-channel uint8 quantization and zlib entropy coding.
  • deserialize_feature_maps() reconstructs feature tensors before the LR-ASPP classifier and object head run.
  • The live payload includes feature shapes and calibration matrices so decoded objects can be placed in global coordinates.
